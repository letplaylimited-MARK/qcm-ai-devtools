📖 用户指南

🧭 Navigator (skill-00) - 意图识别

功能说明

Navigator 负责分析用户输入，识别项目类型、技术偏好和约束条件。

使用示例

from qcm_tools.skills.navigator import Navigator

navigator = Navigator()

# 分析用户意图
intent = navigator.analyze("创建一个高并发的电商后台系统")

# 输出结果
print(f"意图类型: {intent.intent_type}")      # build_custom
print(f"项目类型: {intent.project_type}")    # web_backend
print(f"置信度: {intent.confidence}")       # 0.85

支持的意图类型

类型 说明 示例输入
build_custom 自定义项目构建 "创建一个 FastAPI 项目"
quick_api 快速 API 项目 "快速搭建一个 REST API"
quick_web 快速 Web 项目 "创建一个 Web 应用"

🔍 Scout (skill-03) - 技术选型

功能说明

Scout 根据项目需求，基于七维度评估模型推荐最佳技术栈。

使用示例

from qcm_tools.skills.scout import Scout

scout = Scout()

# 技术选型
tech_result = scout.select(
    requirements={
        "project_type": "web_api",
        "performance": "high",

# 输出结果
print(f"推荐框架: {tech_result.primary_choice}")  # FastAPI
print(f"备选方案: {tech_result.alternatives}")   # ['Flask', 'Django']
print(f"置信度: {tech_result.confidence}")       # 0.88

📋 Planner (skill-04) - 执行规划

功能说明

Planner 将技术选型结果转化为可执行的任务计划，包含优先级和时间估算。

使用示例

from qcm_tools.skills.planner import Planner

planner = Planner()

# 创建执行计划
plan = planner.create_plan(
    tech_stack=["FastAPI", "PostgreSQL", "Redis"],
    project_type="web_api"
)

# 查看任务列表

    print(f"[{task.priority}] {task.name} - 预计: {task.estimated_time}")

✅ Validator (skill-05) - 验收决策

功能说明

Validator 基于五维度验证模型，对项目成果进行质量验收。

使用示例

from qcm_tools.skills.validator import Validator

validator = Validator()

# 执行验收
validation = validator.validate(
    project_path="./my_project",
    criteria={
        "功能完整性": 0.9,
        "文档完整性": 0.8,
        "可执行性": 0.95

# 输出结果
print(f"验收通过: {validation.passed}")      # True
print(f"综合评分: {validation.score}")        # 0.88
print(f"问题列表: {validation.issues}")      # []

🌉 Bridge - ai-skill-system 桥接

功能说明

Bridge 实现 QCM-AI-DevTools 与 ai-skill-system 的双向数据转换。

从 ai-skill-system 导入

from qcm_tools.bridge import create_bridge

bridge = create_bridge()

# 从 YAML 导入交接包
yaml_str = """
schema_version: "1.1"
from_skill: "skill-00"
to_skill: "skill-03"
payload:
  intent_type: "build_custom"
"""

handoff = bridge.import_from_skill_system(yaml_str, format="yaml")
print(f"来源: {handoff.from_skill}")  # skill-00

导出到 ai-skill-system

from qcm_tools.handoff import create_handoff

# 创建交接包
handoff = create_handoff(
    from_skill="skill-05",
    to_skill="skill-00",
    handoff_type="HP-E",
    payload={"validation_result": "passed"}
)

# 导出为 YAML
yaml_output = bridge.export_to_skill_system(handoff, format="yaml")
print(yaml_output)

❓ 常见问题

Q1: QCM-AI-DevTools 与 ai-skill-system 是什么关系？

A: QCM-AI-DevTools 是 ai-skill-system 的代码化执行引擎。ai-skill-system 是一个六层协作 AI 工作流系统，而 QCM-AI-DevTools 将其中的核心 Skills（Navigator、Scout、Planner、Validator）转化为可执行的 Python 代码，实现从"意图"到"交付"的全流程自动化。

Q2: 是否需要 OpenAI API Key？

A: 不强制需要。QCM-AI-DevTools 内置了 Mock 客户端，可以在没有 API Key 的情况下运行。如果需要使用真实的 AI 能力，可以配置 OpenAI API Key。

# 配置 API Key（可选）
import os
os.environ["OPENAI_API_KEY"] = "your-api-key"

Q3: 如何与现有的 ai-skill-system 集成？

A: 通过 Bridge 模块实现无缝集成。Bridge 支持 YAML 格式的双向数据转换，完全兼容 ai-skill-system 的交接包格式。

Q4: 支持哪些项目类型？

A: 目前支持：

  • Web API 项目（FastAPI、Flask、Django）
  • 命令行工具
  • Python 库/包
  • 自定义项目模板

Q5: 测试覆盖率如何？

A: 项目包含 254 个测试用例，覆盖 66% 的核心代码，所有测试全部通过。可以使用以下命令运行测试：

python -m pytest --cov=qcm_tools

🔌 数据库适配器使用

统一 API 操作

from qcm_tools.database import create_adapter, DatabaseConfig, DatabaseType

# 创建适配器（自动识别数据库类型）
config = DatabaseConfig(
    db_type=DatabaseType.POSTGRESQL,  # 或 DatabaseType.MYSQL
    host="localhost",
    database="mydb",
    username="user",
    password="pass"
)
adapter = create_adapter(config)

# 创建表
adapter.create_table("users", {
    "id": "SERIAL PRIMARY KEY",  # PostgreSQL 语法
    "name": "VARCHAR(100) NOT NULL",
    "email": "VARCHAR(255) UNIQUE"
})

# 插入数据
adapter.insert("users", {"name": "Alice", "email": "alice@example.com"})

# 查询数据
users = adapter.select("users", where={"name": "Alice"})

# 更新数据
adapter.update("users", {"email": "new@example.com"}, where={"id": 1})

# UPSERT（PostgreSQL 和 MySQL 都支持）
adapter.upsert("users", 
    data={"id": 1, "name": "Alice", "email": "updated@example.com"},
    conflict_columns=["id"]
)

🔄 连接池配置

from qcm_tools.database import DatabaseConfig, ConnectionPoolConfig

# 自定义连接池配置
pool_config = ConnectionPoolConfig(
    pool_size=10,        # 连接池大小
    max_overflow=20,     # 最大溢出连接数
    pool_timeout=60,     # 获取连接超时（秒）
    pool_recycle=3600,   # 连接回收时间（秒）
    echo=True            # 打印 SQL 语句
)

config = DatabaseConfig(
    db_type=DatabaseType.POSTGRESQL,
    host="localhost",
    database="mydb",
    pool_config=pool_config
)
