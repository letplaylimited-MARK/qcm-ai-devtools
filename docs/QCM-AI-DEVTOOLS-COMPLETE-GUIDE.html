🔌 统一 API 操作

from qcm_tools.database import create_adapter, DatabaseConfig, DatabaseType

# 创建适配器（自动识别数据库类型）
config = DatabaseConfig(
    db_type=DatabaseType.POSTGRESQL,  # 或 DatabaseType.MYSQL
    host="localhost",
    database="mydb",
    username="user",
    password="pass"
)
adapter = create_adapter(config)

# 创建表
adapter.create_table("users", {
    "id": "SERIAL PRIMARY KEY",
    "name": "VARCHAR(100) NOT NULL",
    "email": "VARCHAR(255) UNIQUE"
})

# CRUD 操作
adapter.insert("users", {"name": "Alice", "email": "alice@example.com"})
users = adapter.select("users", where={"name": "Alice"})
adapter.update("users", {"email": "new@example.com"}, where={"id": 1})
adapter.delete("users", where={"id": 1})

# UPSERT（PostgreSQL 和 MySQL 都支持）
adapter.upsert("users", 
    data={"id": 1, "name": "Alice"},
    conflict_columns=["id"]
)

🔌 OpenCode 使用示例

from qcm_tools.ai.providers import (
    AIProviderConfig,
    AIProviderType,
    AIMessage,
    create_provider
)

# 配置 OpenCode
config = AIProviderConfig(
    provider_type=AIProviderType.OPENCODE,
    base_url="http://localhost:8080/v1",  # OpenCode 服务地址
    model="gpt-4"
)

provider = create_provider(config)

# 发送消息
response = provider.chat([
    AIMessage(role="user", content="你好！")
])
print(response.content)

# 切换模型
provider.set_model("claude-3-opus")  # 切换到 Claude
provider.set_model("deepseek-chat")   # 切换到 DeepSeek

# 查看模型后端
print(provider.get_model_backend())  # "anthropic"

❓ 常见问题

快速解答您的疑问

Q: QCM-AI-DevTools 与 ai-skill-system 是什么关系？

A: QCM-AI-DevTools 是 ai-skill-system 的代码化执行引擎。ai-skill-system 是六层协作 AI 工作流系统，QCM-AI-DevTools 将其中的核心 Skills 转化为可执行的 Python 代码。

Q: 是否需要 OpenAI API Key？

A: 不强制需要。内置 Mock 客户端可在无 API Key 情况下运行。支持多种 AI 提供者，可使用 OpenCode 接入不同模型。

Q: 支持哪些数据库？

A: 支持 PostgreSQL、MySQL 和 SQLite。提供统一 API，可自由切换数据库类型。

Q: 如何与 OpenCode 集成？

A: 使用 OpenCodeProvider，配置 OpenCode 服务地址即可。支持自由切换不同 AI 模型。

Q: 测试覆盖率如何？

A: 项目包含 291+ 测试用例，覆盖 66%+ 核心代码，所有测试全部通过。
