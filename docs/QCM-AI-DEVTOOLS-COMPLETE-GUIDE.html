🔄 数据流转机制

Handoff Package（交接包）是 Skills 之间传递上下文的核心机制：

# 交接包结构 (YAML 格式)
handoff:
  schema_version: '1.1'
  from_skill: skill-00        # 来源 Skill
  to_skill: skill-03          # 目标 Skill
  handoff_type: HP-D          # 交接包类型
  payload:                      # 数据负载
    intent_type: find_open_source
    confidence_score: 0.85
    project_summary: "开发一个情感分析系统"
    core_requirements:
      - "支持中文分词"
      - "情感倾向分析"
  self_review:                  # 自我审查 (v1.1)
    assumptions:
      - "用户需要 Python 解决方案"
    potential_failures:
      - "开源库可能不支持特定领域"
  downstream_notes:             # 下游注意事项
    to_skill: skill-03
    cautions:
      - "关注中文支持能力"

📦 交接包类型体系

类型 名称 流转路径 用途
HP-A 提示词优化包 S01 → S02 传递优化后的提示词
HP-B 技术选型包 S03 → S02 传递技术选型结果
HP-C 工程包交付 S02 → S05 传递工程包元数据
HP-D 路由推荐包 S00 → 任意 传递意图分析结果
HP-E 操作手册 S04 → 用户 传递执行计划
HP-F 验收报告 S05 → 发布 传递验收结果

from qcm_tools.skills import Scout

async def select_tech():
    scout = Scout()
    
    # 对比多个框架
    comparison = await scout.compare_libraries(
        ["fastapi", "flask", "django"],
        requirement="构建高性能 RESTful API"
    )
    
    print(f"推荐方案: {comparison.winner}")
    print(f"综合评分: {comparison.comparison_matrix[comparison.winner]['overall']:.1f}")

asyncio.run(select_tech())

🔌 统一 CRUD API

from qcm_tools.database import create_adapter, DatabaseConfig, DatabaseType

# 创建适配器（自动识别数据库类型）
config = DatabaseConfig(
    db_type=DatabaseType.POSTGRESQL,  # 或 DatabaseType.MYSQL
    host="localhost",
    database="mydb",
    username="user",
    password="pass"
)
adapter = create_adapter(config)

===== 建表操作 =====
adapter.create_table("users", {
    "id": "SERIAL PRIMARY KEY",
    "name": "VARCHAR(100) NOT NULL",
    "email": "VARCHAR(255) UNIQUE"
})

===== 插入数据 =====
adapter.insert("users", {"name": "Alice", "email": "alice@example.com"})

===== 查询数据 =====
users = adapter.select("users", where={"name": "Alice"})

===== 更新数据 =====
adapter.update("users", {"email": "new@example.com"}, where={"id": 1})

===== 删除数据 =====
adapter.delete("users", where={"id": 1})

===== UPSERT（PostgreSQL 和 MySQL 都支持）=====
adapter.upsert("users", 
    data={"id": 1, "name": "Alice"},
    conflict_columns=["id"]
)

🔍 Scout 使用指南

职责：基于七维度评估模型进行技术选型

import asyncio
from qcm_tools.skills.scout import Scout

async def main():
    scout = Scout()
    
    ===== 评估单个库 =====
    evaluation = await scout.evaluate_library(
        "fastapi",
        requirement="构建高性能 RESTful API",
        constraints=["Python 3.8+", "异步支持"]
    )
    
    print(f"综合评分: {evaluation.overall_score:.1f}")
    print(f"推荐级别: {evaluation.recommendation}")
    print(f"优势: {evaluation.pros}")
    print(f"劣势: {evaluation.cons}")
    
    ===== 对比多个库 =====
    comparison = await scout.compare_libraries(
        ["fastapi", "flask", "django"],
        requirement="构建 RESTful API"
    )
    
    print(f"推荐方案: {comparison.winner}")
    print(f"对比摘要: {comparison.summary}")
    
    ===== 智能推荐最佳选择 =====
    best = await scout.recommend_best(
        requirement="构建高性能 RESTful API",
        language="python",
        limit=3
    )
    print(f"最佳推荐: {best.library.name}")

asyncio.run(main())

❓ 常见问题

FAQ

快速解答您的疑问

Q: QCM-AI-DevTools 与 ai-skill-system 是什么关系？

A: QCM-AI-DevTools 是 ai-skill-system 的代码化执行引擎。ai-skill-system 是六层协作 AI 工作流系统，QCM-AI-DevTools 将其中的核心 Skills（Navigator、Scout、Planner、Validator）转化为可执行的 Python 代码，并实现了标准化的交接包机制。

Q: 是否需要 OpenAI API Key？

A: 不强制需要。内置 Mock 客户端可在无 API Key 情况下运行。支持多种 AI 提供者，可使用 OpenCode 接入不同模型，或使用 Ollama 运行本地模型。

Q: 支持哪些数据库？

A: 支持 PostgreSQL、MySQL 和 SQLite 三种数据库。提供统一 API，可在三种数据库间自由切换，无需修改业务代码。

Q: 如何与 OpenCode 集成？

A: 使用 OpenCodeProvider，配置 OpenCode 服务地址即可。支持自由切换不同 AI 模型（GPT-4、Claude、DeepSeek、Gemini 等）。

Q: 测试覆盖率如何？

A: 项目包含 291+ 测试用例，覆盖 66%+ 核心代码，所有测试全部通过。可运行 python -m pytest 验证。

Q: 如何报告问题或贡献代码？

A: 请访问 GitHub 仓库：
github.com/letplaylimited-MARK/qcm-ai-devtools
可提交 Issue 报告问题或 Pull Request 贡献代码。

Q: 交接包（Handoff Package）是什么？

A: 交接包是 Skills 之间传递上下文的标准化数据结构，采用 YAML 格式。包含来源 Skill、目标 Skill、数据负载、置信度评分等信息，确保信息在 Skill 间无损传递。

Q: 支持异步操作吗？

A: 支持。数据库模块支持 asyncpg（PostgreSQL）和 aiomysql（MySQL）异步驱动。AI 提供者支持 achat() 异步对话方法。
